# 需求文档

## 1. 应用概述

### 1.1 应用名称
AI Coding平台

### 1.2 应用描述
一个面向开发团队的AI辅助编码平台,采用多租户架构设计。平台提供全局Skill市场和Prompt市场,租户内包含智能会话、数据大盘、需求分析、智能评审、智能测试及空间设置等核心功能模块,支持与meego需求管理平台和gitlab代码仓库集成。平台以智能会话为核心交互方式,通过统一的对话界面完成UI设计、需求分析、代码编写等任务。平台支持超级管理员、租户管理员、普通用户三种角色模式。

## 2. 用户与使用场景

### 2.1 目标用户
- 超级管理员（平台管理者）
- 租户管理员（团队管理者）
- 开发团队成员（开发者、设计师、产品经理、测试人员）

### 2.2 核心使用场景
- 平台管理者进行全局配置与空间管理
- 团队协作进行软件开发全流程管理
- 利用AI能力提升开发效率
- 统一团队编码规范和设计规范
- 跨团队共享Skill和Prompt资源

### 3.2 登录页

- 提供账号密码登录
- 支持三种角色登录模式：
  + 使用superadmin登录进入超级管理员模式
  + 使用admin登录进入租户管理员模式

**轮播标语**

  + 全栈协同，多端智研
  + 规范统驭，提质增效
  + 虾班智守，下班无忧

### 3.5 租户工作空间功能

#### 3.5.1 智能会话（主界面）

**会话界面布局**
- 左侧：会话历史列表
  + 展示所有历史会话记录
  + 支持按时间排序
  + 支持搜索会话内容
  + 支持新建会话
  + 支持删除会话
- 中间：当前会话内容区
  + 展示当前会话的完整对话记录
  + 底部提供居中显示的大型对话输入框
  + 输入框下方包含功能选择按钮：代码、技能等
  + 技能选项包含：需求设计、UI设计、代码编写、测试编写、需求上线

**核心功能**
- 通过对话完成UI设计需求描述,AI生成UI设计方案
- 通过对话创建需求,AI生成需求文档
- 新建需求后自动跳转到智能会话页面,输入框中自动填充「#需求设计」并@对应的需求
- 通过对话将需求转换为spec文档
- 通过对话进行代码编写,AI提供代码补全和代码建议
- 支持在会话中调用空间已安装的Skill
- 支持在会话中使用Prompt
- 支持将需求同步到meego需求管理平台
- 支持保存代码到gitlab

**会话上下文管理**
- 会话内容自动保存
- 支持查看历史会话记录
- 支持在历史会话基础上继续对话

**需求管理功能**
- 在会话中创建的需求自动归档
- 支持查看需求列表（看板模式/列表模式）
  + 看板模式：以看板形式展示需求（待处理、进行中、已完成等状态）,支持拖拽需求卡片切换状态,每个状态列头部展示对应的颜色标识
  + 列表模式：以列表形式展示需求,展示需求详细信息（标题、状态、负责人、创建时间）,支持筛选和排序
- 支持点击需求进入详情页面
  + 详情页面展示需求完整信息
  + 详情页面提供「上一个」和「下一个」按钮,支持快速切换查看相邻需求
  + 详情页面中状态字段使用下拉框形式,支持直接修改状态

**代码管理功能**
- 在会话中编写的代码自动保存
- 支持查看代码文件列表
- 支持在线编辑代码
- 支持点击代码文件进入对应会话查看编写过程

**工程代码库类型选择**
- 支持选择仓库类型：开发库、用例库（默认开发库）
- 用例库没有预览模式
- 同步分支功能以图标按钮形式展示

#### 3.5.2 数据大盘
- 展示代码提交数量统计（按时间维度）
- 展示会话数量统计（按时间维度）
- 展示需求完成数统计（按时间维度）
- 展示每个成员的会话轨迹（包括会话时间、会话主题、会话时长等信息）
- 支持点击成员会话轨迹进入详情页面
  + 详情页面以信息流形式展示单个成员的会话轨迹
  + 提供「会话复盘」按钮
- 使用图表形式可视化展示统计数据

#### 3.5.3 需求分析

**列表模式**
- 以列表形式展示所有需求
- 展示需求详细信息（标题、状态、负责人、创建时间、优先级）
- 支持按状态、负责人、创建时间等条件筛选需求
- 支持按字段排序
- 支持点击需求查看详情

**看板模式**
- 以看板形式展示需求（待处理、进行中、已完成等状态）
- 每个状态列头部展示对应的颜色标识
- 支持拖拽需求卡片切换状态
- 需求卡片展示标题、负责人、优先级等关键信息
- 支持点击需求卡片查看详情

**需求详情页面**
- 展示需求完整信息
- 提供「上一个」和「下一个」按钮,支持快速切换查看相邻需求
- 状态字段使用下拉框形式,支持直接修改状态

**模式切换**

- 支持暗色和亮色主题切换（前景色和背景色同时生效）
- markdown内容展示格式化后的效果
- 左上角提供切换raw模式按钮，展示源文件内容（参考github展示方案）

#### 3.5.5 智能测试
- 提交代码或功能进行智能测试
- AI辅助生成测试用例
- AI执行测试并展示测试结果

- 配置项目绑定的meego项目信息

- 支持设置规范，点击后弹出工程规范、设计规范的弹窗（tab页形式，使用markdown编辑器）

**研发规范**
- 使用二级tab结构：
  + 编码规范（使用markdown编辑器）
  + 设计规范（使用markdown编辑器）

**空间Skill**
- 按开发阶段分类展示Skill配置

  + 需求设计：提供技能下拉框,支持配置技能
  + UI设计：提供技能下拉框,支持配置技能
  + 代码开发：提供技能下拉框,支持配置技能
  + 测试编写：提供技能下拉框,支持配置技能
  + 需求上线：提供技能下拉框,支持配置技能
  + 其他从需求设计到上线的所有环节：提供技能下拉框,支持配置技能
- 支持从Skill市场添加Skill
- 支持移除Skill

**空间Prompt**
- 按开发阶段分类展示Prompt
- 以卡片形式展示Prompt列表
- 每个Prompt卡片支持复制功能
- 支持从Prompt市场添加Prompt
- 支持移除Prompt

**智能体设置**
- 配置空间内AI智能体参数
- 配置智能体行为规则

**成员管理**
- 展示空间成员列表（姓名、角色、加入时间）
- 支持添加成员到空间
- 支持移除成员
- 支持设置成员角色权限

**权限控制**
- 用户模式下空间设置仅可查看不可编辑
- 租户管理员模式下可编辑空间设置

### 3.6 个人资料页
- 展示用户个人信息
- 支持编辑个人资料

### 3.7 退出登录
- 提供退出登录功能
- 退出后返回登录页

## 4. 业务规则与逻辑

### 4.1 角色权限规则
- 超级管理员：登录后进入管理后台，可进行全局配置与空间管理
- 租户管理员：登录后进入租户工作空间，可编辑空间设置
- 普通用户：登录后进入租户工作空间，空间设置仅可查看不可编辑

### 4.2 多租户隔离规则
- 每个租户拥有独立的工作空间
- 租户间数据完全隔离,不可跨租户访问
- 租户内成员仅可访问所属租户的数据和功能

### 4.3 Skill和Prompt使用规则
- Skill需先从市场安装到空间才可使用
- Prompt可直接从市场使用,也可添加到空间常用列表
- 空间内Skill和Prompt配置仅对当前租户生效
- 在智能会话中可直接调用已安装的Skill和Prompt

### 4.4 会话与任务关联规则
- 所有UI设计、需求分析、代码编写任务均在智能会话中完成
- 每个任务自动关联到对应会话记录
- 支持通过会话历史追溯任务完成过程
- 新建需求后自动跳转到智能会话页面,输入框中自动填充「#需求设计」并@对应的需求

### 4.5 需求同步规则
- 智能会话中创建的需求可同步到meego需求管理平台
- 需求分析模块支持从meego需求管理平台同步需求到平台
- 同步时需使用空间设置中配置的meego项目信息
- 同步后需求在平台和meego中保持关联

### 4.6 代码编辑与评审规则
- 智能会话中编写的代码可提交到智能评审模块
- 智能评审基于空间设置中的编码规范进行评审
- 评审结果以PR记录列表形式展示
- 点击PR记录可查看详细评审结果

### 4.7 仓库类型规则
- git仓库支持两种类型：开发库、用例库
- 默认仓库类型为开发库
- 用例库不提供预览模式
- 工程代码中可选择仓库类型

### 4.8 数据统计规则

**租户数据大盘统计规则**
- 数据大盘统计范围为当前租户空间内的数据
- 代码提交数来源于gitlab集成数据
- 会话数量统计智能会话模块中的所有会话次数
- 需求完成数统计智能会话中创建的已完成状态需求数量
- 成员会话轨迹记录每个成员在智能会话中的活动信息
- 支持点击成员会话轨迹进入详情页面查看单个成员的会话信息流
- 会话详情页面提供会话复盘功能

**超级管理员数据大盘统计规则**
- AI完成需求总量统计平台所有空间中AI完成的需求数量
- AI完成需求分布按空间维度统计各空间的AI完成需求数量
- AI完成需求趋势按时间维度统计AI完成需求数量的变化
- 会话总数统计平台所有空间的会话总数
- 会话分布按空间维度统计各空间的会话数量,按云侧和端侧维度统计会话数量
- 会话趋势按时间维度统计会话总数的变化
- token总消耗量统计平台所有会话消耗的token总量
- token消耗趋势按时间维度统计token消耗量的变化
- 产品、开发、测试会话token消耗占比统计三类会话各自消耗的token占总消耗量的比例
- 产品、开发、测试会话token消耗占比趋势按时间维度统计三类会话token消耗占比的变化

### 4.9 需求/用例/缺陷看板规则
- 看板模式中每个状态列头部展示对应的颜色标识,用于快速区分不同状态
- 需求详情页面提供「上一个」和「下一个」按钮,按当前列表顺序切换查看相邻需求
- 需求详情页面中状态字段使用下拉框形式,支持直接选择修改状态

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| Skill安装失败 | 提示安装失败原因,允许重试 |
| meego同步失败 | 提示同步失败,需求保留在平台,支持重新同步 |
| gitlab连接失败 | 提示连接失败,检查配置信息 |
| 非空间成员访问租户数据 | 拒绝访问,提示无权限 |
| 会话保存失败 | 提示保存失败,保留本地缓存 |
| AI服务响应超时 | 提示服务繁忙,建议稍后重试 |
| 空间设置配置项为空 | 提示必填项未配置,相关功能不可用 |
| 会话历史记录过多 | 支持分页加载或按时间范围筛选 |
| 普通用户尝试编辑空间设置 | 拒绝操作,提示无编辑权限 |
| 超级管理员配置权限限制后租户管理员尝试修改 | 拒绝操作,提示该配置项已被锁定 |
| 需求详情页面切换到列表边界 | 到达第一个或最后一个需求时,对应的「上一个」或「下一个」按钮置灰不可点击 |

## 6. 验收标准

1. 使用superadmin登录平台,进入超级管理员后台的数据大盘页面,查看AI完成需求总量、分布、趋势,查看会话总数、分布、趋势,查看token总消耗量、趋势及产品开发测试会话占比趋势
2. 进入超级管理员后台的全局配置页面,查看规范设置、CICD设置、智能体设置三个子页面
3. 使用admin登录平台,进入租户工作空间的需求分析模块,切换到看板模式,确认每个状态列头部展示对应颜色标识
4. 点击需求卡片进入详情页面,确认状态字段为下拉框形式,点击「下一个」按钮切换到下一个需求
5. 进入空间设置的成员管理页面,点击「添加成员」按钮添加成员
6. 点击个人资料页面,查看并编辑个人信息
7. 点击退出登录,确认返回登录页
